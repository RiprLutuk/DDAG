# Support

## Usage and architecture questions

Use [GitHub Discussions](https://github.com/RiprLutuk/DDAG/discussions) for setup help, architecture questions, connector discussions, and deployment guidance.

## Bugs

Use the [bug report form](https://github.com/RiprLutuk/DDAG/issues/new?template=bug_report.yml) with reproduction steps, version/commit, deployment type, and redacted structured logs. A DDAG `request_id` is especially useful for tracing.

## Feature requests

Use the [feature request form](https://github.com/RiprLutuk/DDAG/issues/new?template=feature_request.yml). Explain the user problem, expected outcome, and relevant database or deployment constraints.

## Security

Report vulnerabilities privately through [GitHub Security Advisories](https://github.com/RiprLutuk/DDAG/security/advisories/new). Never post credentials, tokens, private keys, customer data, or exploitable details publicly.

Community support is best-effort; no response-time or uptime SLA is currently offered.
